Beste Ordina collega's,

Bij deze wilde ik jullie informeren over een prive project waar ik momenteel bezig ben. Het project heet de RelayConnector en is een open source component die voor het Cordys platform gebruik kan worden. Zoals de meesten van jullie wellicht weten ben ik een groot voorstander van open source software en vin dik het als hobby leuk om te programmeren. Hierbij vind ik het vooral leuk als ik zelf ontwikkelde programmatuur kan toepassen, en als er ook andere gebruikers zijn. 

In het geval van de RelayConnector wordt deze momenteel in een aantal projectjes van de IB-Groep in gebruikt. Ik liep al langer rond met ideeen voor krachtigere transformatie mogelijkheden en meer flexibiliteit voor de As400Connector. Dit was puur omdat ik dacht dat het mooier en beter kon, en niet omdat er een opdracht toe was. In januari kwam er in een project een heel specifiek probleem voor waarvoor getwijfeld werd of Cordys daarvoor gebruikt kon worden. Met de bestaande middelen (short-lived BPM of maatwerk java) kon dit wel, maar niet heel goed of elegant. Met name kon short-lived BPM niet parallel meerdere services tegelijk aanroepen, terwijl performance heel belangrijk was. Ik heb toen in een weekend de ideeen die ik had omgezet in de RelayConnector, die ik van scratch heb opgebouwd. Die reayConnector heb ik zo opgezet dat dit specifieke probleem er mee aangepakt kon worden, maar ook herbruikbaar zou zijn voor andere doeleinden.

Sindsdien blijkt de RelayConnector voor tal van situaties een heel handige oplossing te bieden, en gebruiken de andere Cordys ontwikkelaars van de IB-Groep (Edde van der Vosse en Joost Top) hem op kleine schaal in andere projecten. Daarnaast heb ik nog een aantal ideeen waarmee vrij eenvoudig de software uitgebreid kan worden voor andere toepassingen. Bij de IB-Groep hebben we het gebruik hiervan besproken met architecten in een overleg over ontwikkeltools voor services. Daarbij hebben we de vrijheid gekregen om als ontwikkelaars tesamen de tools te kiezen die wij het best achten, waarbij het gebruik van de RelayConnector expliciet niet uitgesloten is.

Vanaf het eerste idee wist ik dat ik een dergelijk product als open source zou willen ontwikkelen. Vandaar dat ik heel bewust heb gekozen om alle software in mijn eigen tijd thuis te ontwikkelen, en niet in tijd betaald door de IB-Groep of door Ordina. Juist om meer tijd voor dit soort projecten te hebben werk ik al lange tijd maar 80%, hoewel ik er ook regelmatig aan werk in de avonden en weekenden. Hieronder vallen ook het documenteren en testen. Door het gebruik bij de IB-Groep komt er wel af en toe een fout of een nieuwe wens naar boven. Deze problemen los ik dan thuis op, hoewel de source code ook bij de IB-Groep beschikbaar is (dat is immers belangrijk bij open source). 

Ik ben voorzichtig geweest om geen code te gebruiken uit de As400Connector of andere software, waarvan de ontwikkeling is betaald door de IB-Groep. Ook maak ik geen gebruik van code van Cordys (anders dan af en toe een voorbeeldregel uit de documentatie hoe de Cordys API werkt), en als ik het project publiek beschikbar stel zal ik zorgen dat hier geen Cordys libraries of andere mogelijk beschermde software bij zit. Die heb ik nu ook al buiten het versie beheer gehouden. 

Het ligt in mijn bedoeling het project binnenkort op Internet publiekelijk beschikbaar te stellen. De precieze details hoe en waar (Sourceforge, Cordys Wiki, eigen prive servers) weet ik nog niet. Het is primair de bedoeling om de source code en handleiding(en) inclusief de versiebeheer historie (een git respository) beschikbaar te stellen. Wellicht dat er ook vanuit andere Cordys klanten interesse is. Dit kunnen bestaande of potentiele klanten van Ordina zijn, maar ook van concurrenten, want onder een open source licentie is iedereen vrij de code te gebruiken. Ik verwacht overigens niet dat dit meteen zo'n vaart zal lopen, en verwacht dat de relayConnector voorlopig alleen bij de IB-Groep gebruikt zal worden.

Ik meld dit allemaal aan jullie als Ordina collega's om verschillende redenen. Ten eerste om reclame te maken voor iets waar ik met veel plezier aan werk, en waarvan ik denk dat het in vele situaties nuttig gebruikt kan worden. Ten tweede omdat de IB-Groep een klant van Ordina is. Ten derde omdat Cordys een partner is van Ordina. Tot slot omdat ik een medewerker ben van Ordina. De naam van Ordina heb ik op geen enkele wijze in verband gebracht met dit project. Mocht dat (b.v. in presentaties of publicaties) wel gewenst zijn, dan sta ik daar zeker voor open. Het copyright zal echter bij mij blijven en als Java package naam wordt ook kisst.org gebruikt, wat mijn persoonlijke domein naam is voor. De domeinnaam kisst staat voor Keep It Simple Software Technology, wat ook voor de RelayConnector opgaat, en de org TLD is de standaard voor alle open source projecten.

Ik heb ook het voornemen om, net als jullie van Ordina, ook Cordys op de hoogte te stellen van dit project en mijn voornemens (informeel heb ik het al wel eens besproken met Mark van der Wal). Aangezien Ordina een partner is van Cordys, kan ik mij voorstellen dat er vanuit alliance management misschien wensen of suggesties zijn hoe dit te communiceren. Zo niet dan zal ik een vergelijkbare mail als deze naar Mark van der Wal sturen, aangezien hij mijn voornaamste aanspreekpunt is binnen Cordys.

Tot slot heb ik ter promotie en beeldvorming de voorlopig handleiding toegevoegd aan deze mail. Deze handleiding is nog een work in progress, en eerder een Reference Manual dan een lekker leesbaar document. Voor dat laatste hoop ik snel een soort CookBook met voorbeeldjes te maken. Overigens zijn de ideeen van de RelayConnector niet heel specifiek voor Cordys. De technische implementatie is dat wel aangezien deze gebruik maakt van de Cordys NOM XML libraries. Het lijkt me wel leuk om te kijken of iets vergelijkbaars ook zinvol is voor ESB omgevingen van andere leveranciers.

Ik hoop jullie hiermee voldoende geinformeerd te hebben. Mochten jullie nog vragen of suggesties hebben, dan hoor ik dat graag. Ik vertel natuurlijk ook graag meer over de ideeen, toepassingen, werking of plannen van de RelayConnector.

Groeten,

Mark Hooijkaas
